# 网站部署指南

本文档说明如何将个人网站部署到不同的平台。

## 📦 本地运行

### 最简单的方法
1. 双击 `index.html` 文件
2. 在默认浏览器中打开

### 使用本地服务器（推荐）
```bash
# 方法1：使用Python
python -m http.server 8000

# 方法2：使用Node.js（需要安装http-server）
npm install -g http-server
http-server
```

访问 `http://localhost:8000` 查看网站。

## 🌐 在线部署

### 1. GitHub Pages（免费）

#### 步骤：
1. 创建GitHub账号（如果没有）
2. 新建仓库，命名为 `username.github.io`（username为您的GitHub用户名）
3. 将所有网站文件上传到仓库
4. 进入仓库设置 → Pages → 选择 main 分支
5. 等待几分钟，访问 `https://username.github.io`

#### 注意事项：
- 确保HTML文件在根目录
- 使用相对路径引用CSS/JS文件
- 图片等资源文件一并上传

### 2. Netlify（免费，推荐）

#### 步骤：
1. 访问 [netlify.com](https://netlify.com) 注册账号
2. 拖拽 `personal_website` 文件夹到部署区域
3. 自动生成部署链接
4. 可自定义域名

#### 优点：
- 自动HTTPS
- 全球CDN
- 持续部署
- 自定义域名

### 3. Vercel（免费）

#### 步骤：
1. 访问 [vercel.com](https://vercel.com) 注册账号
2. 导入GitHub仓库或拖拽文件
3. 自动部署并生成链接

## 🔧 自定义域名

### 购买域名
- 阿里云
- 腾讯云
- Namecheap
- GoDaddy

### 配置步骤：
1. 在域名注册商处添加DNS记录
2. 指向您的部署平台（GitHub Pages/Netlify等）
3. 在部署平台绑定自定义域名
4. 等待DNS生效（最多48小时）

## 🚨 故障排除

### 常见问题：

#### 1. 图片不显示
- 检查图片路径是否正确
- 确认图片文件是否存在
- 检查文件权限

#### 2. CSS样式不生效
- 检查CSS文件路径
- 查看浏览器开发者工具
- 清除浏览器缓存

#### 3. JavaScript错误
- 检查控制台错误信息
- 验证JS语法
- 确保DOM元素存在

#### 4. 响应式问题
- 检查视口meta标签
- 验证媒体查询
- 测试不同设备

### 调试工具：
- Chrome/Firefox开发者工具
- 响应式设计模式
- 网络面板
- 控制台
